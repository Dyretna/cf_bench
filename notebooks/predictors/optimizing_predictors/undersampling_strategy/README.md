# Undersampling Strategy Experiments

This folder contains notebooks investigating the impact of training data balancing strategies on both model performance and counterfactual generation quality.

## Overview

These experiments compare Random Forest classifiers trained on:
1. **Undersampled (balanced) data**: Majority class examples randomly deleted to achieve 1:1 class ratio
2. **Original (imbalanced) data**: Natural class distribution preserved

The goal is to understand whether physically removing majority class samples affects model behavior and counterfactual generation quality.

## Terminology & Approach Clarification

### What is Undersampling?

**Undersampling** is a data-level technique where majority class samples are **physically removed** from the training dataset to balance class frequencies. In these experiments:
- Target class `0` (negative/no disease): ~90% of original data
- Target class `1` (positive/disease): ~10% of original data
- **Undersampling action**: Randomly delete `0` samples until count(0) ≈ count(1)
- **Result**: Balanced training set with 50/50 class distribution

**Key characteristic**: The actual training dataset is smaller and no longer represents the population distribution.

### Undersampling vs Class Weighting

These are **fundamentally different** approaches to handling imbalanced data:

| Aspect | Undersampling (Data-level) | Class Weighting (Algorithm-level) |
|--------|---------------------------|-----------------------------------|
| **Data modification** | Physically removes samples | Keeps all samples |
| **Training set size** | Smaller (discards majority data) | Original size (no data loss) |
| **Population representation** | No longer representative | Maintains population distribution |
| **Mechanism** | Changes what model sees | Changes how model values errors |
| **Implementation** | Pre-processing step | Model hyperparameter (`class_weight="balanced"`, `scale_pos_weight`) |
| **Information loss** | Loses potentially useful majority class patterns | Retains all information |

**Example:**
- Original data: 9000 class 0, 1000 class 1
- **Undersampling**: Keep all 1000 class 1, randomly delete 8000 class 0 → Train on 2000 samples
- **Class weighting**: Keep all 10,000 samples, but weight class 1 errors 9× higher during optimization

### Why This Matters for Counterfactual Generation

**Population Representativeness Problem:**

When you undersample, the model learns from a 50/50 distribution that doesn't exist in reality. This creates several issues:

1. **Base rate mismatch**: The model's internal probability calibration assumes equal class frequencies, but real-world predictions happen on 90/10 data.

2. **Feature distribution shift**: Discarding 80% of majority class samples means the model may never see important feature combinations or edge cases that exist in the population.

3. **Counterfactual realism**: CFs generated by an undersampled model might suggest changes that work in the balanced training space but are unrealistic for the actual population distribution.

**Levels of Impact:**

**Level 1 - Statistical Validity:**
- Undersampled models have poorly calibrated probabilities
- A prediction of "50% risk" from an undersampled model doesn't mean 50% risk in the real population
- This affects CF validity: meeting a target probability threshold may not have the intended real-world meaning

**Level 2 - Feature Space Coverage:**
- Majority class contains important variation that gets lost
- Some feature combinations only appear in discarded samples
- CFs might miss more natural/common pathways because those patterns were deleted

**Level 3 - Scientific Validity:**
- The model no longer represents population-level relationships
- Conclusions about "what changes reduce risk" apply to the artificial 50/50 world, not reality
- Generalization to new data is compromised

**Level 4 - Fairness/Equity:**
- If certain subgroups are concentrated in the majority class, undersampling disproportionately removes their representation
- CFs may work differently for underrepresented vs well-represented populations

### [02_cfs_on_balanced_vs_unbalanced.ipynb](02_cfs_on_balanced_vs_unbalanced.ipynb)

**Purpose**: Compare counterfactual generation quality between models trained on balanced vs unbalanced data.

**Key aspects**:
- Loads counterfactual outputs from both balanced and unbalanced model runs
- Compares four configurations:
  - Balanced data + Random CF generation
  - Unbalanced data + Random CF generation
  - Balanced data + Genetic CF generation
  - Unbalanced data + Genetic CF generation
- Implements `clean_counterfactuals()` function to remove unchanged features (sets them to NaN for clarity)
- Calculates success metrics:
  - Total counterfactuals generated
  - Number meeting target risk threshold
  - Success rate
  - Mean original risk

**Data sources**:
- `CF_OUTPUTS/balanced_04_09/`: Counterfactuals from balanced training
- `CF_OUTPUTS/unbalanced_04_09/`: Counterfactuals from unbalanced training

**Analysis focus**: Determining whether the training data balance affects:
- Counterfactual validity (meeting target risk)
- Feature change patterns
- Overall CF generation success rates

**Critical interpretation note**: If undersampled models generate more "valid" CFs (meeting target threshold), this could indicate:
- ✓ Better exploration of the decision boundary
- ✗ Overfitting to the artificial 50/50 distribution
- ✗ CFs that work in training space but not in real population

The **validity** of a CF must be considered in context of the population it will be applied to.

## Research Questions Addressed

These experiments investigate:

1. **Performance comparison**: Does undersampling improve minority class recall without excessive false positive rates?

2. **CF quality**: Do models trained on balanced data generate more valid/achievable counterfactuals?

3. **CF characteristics**: How does training distribution affect the types of feature changes suggested?

4. **Population relevance**: Are CFs from undersampled models still applicable to real-world (imbalanced) populations?

## Key Findings & Implications

### Decision Boundary Shifts

Both undersampling and class weighting shift the decision boundary toward the minority class, but through different mechanisms:

- **Undersampling**: Model learns from equal class frequencies, naturally places boundary near the middle of the feature space
- **Class weighting**: Model sees true distribution but is told to care more about minority errors

**Shared outcome**: Both reduce false negatives (improve recall) but increase false positives.

### When Undersampling Might Be Justified

Despite the representativeness concerns, undersampling can be appropriate when:

1. **Majority class is highly redundant**: If 90% of majority samples are near-duplicates, keeping 10% preserves all information
2. **Computational constraints**: Smaller dataset enables faster experimentation
3. **Exploratory phase**: Quick way to see if balance helps before investing in more sophisticated approaches
4. **Ensemble member**: One undersampled model combined with others in a voting scheme

### When Undersampling Is Problematic

Undersampling should be avoided when:

1. **Majority class has rich variation**: Diverse patterns get lost
2. **Real probability calibration matters**: For risk prediction or decision-making
3. **Small dataset**: Can't afford to discard 80% of data
4. **Production deployment**: Model must handle real-world distribution
5. **Scientific inference**: Need to understand true population relationships

### Alternatives to Consider

Instead of undersampling, consider:

1. **Class weighting**: `class_weight="balanced_subsample"` or `scale_pos_weight`
2. **SMOTE/oversampling**: Synthesize minority samples instead of deleting majority
3. **Cost-sensitive learning**: Explicitly specify misclassification costs
4. **Threshold adjustment**: Train on imbalanced data, tune decision threshold post-hoc
5. **Ensemble methods**: Combine multiple models trained on different subsamples

## Recommendations

Based on these experiments:

### For Model Training
- **Prefer class weighting over undersampling** for production models
- Use undersampling only for rapid prototyping or computational constraints
- Always evaluate on test data that reflects real population distribution

### For Counterfactual Generation
- **Test CF validity on both distributions**: Do CFs from undersampled models transfer to real data?
- Compare CF "naturalness": Are suggested changes realistic given population statistics?
- Consider ensemble approach: Generate CFs from both model types and evaluate quality

### For Research Interpretation
- **Report both training and population distributions** clearly
- Acknowledge when findings apply to balanced training space vs real-world
- Consider probability calibration methods (Platt scaling, isotonic regression) to bridge the gap
